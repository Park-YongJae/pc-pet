# PC-PET Product Requirements Document

## 개요

PC-PET은 Windows 및 macOS 데스크탑 위에 떠다니는 가상 펫 애플리케이션입니다. 투명 오버레이 방식으로 항상 화면 최상위에 표시되며, 클릭 반응·먹이주기·스트레스 관리·Claude AI 대화 기능을 갖춥니다.

---

## 1. 펫 생애 주기

### 시작 (알 시스템)
- **최초 실행 시**: 화면 중앙에 알 표시
- **펫 사망 후**: 동일하게 알 화면으로 전환
- 알을 클릭하면 균열 애니메이션 → 10종 중 랜덤 1종 부화

### 사망 조건
| 조건 | 설명 |
|---|---|
| 배고픔 = 0 | 굶어 죽음 |
| 스트레스 = 100 | 스트레스 과다로 죽음 |

- 사망 후 부활 없음 → 새 알에서 다시 시작
- `total_deaths` 누적 기록

## 3. 캐릭터 상태 기계 (FSM)

```
EGG ──클릭──▶ HATCHING ──완료──▶ IDLE

IDLE ──클릭 1~3회──▶ REACT_HAPPY ──완료──▶ IDLE
IDLE ──클릭 4~7회──▶ REACT_ANNOYED ──완료──▶ IDLE
IDLE ──클릭 8회+──▶ REACT_ANGRY ──완료──▶ IDLE
IDLE ──랜덤 타이머──▶ WALKING ──도착──▶ IDLE
IDLE ──배고픔<70──▶ HUNGRY
HUNGRY ──먹이주기──▶ EATING ──완료──▶ IDLE
IDLE/HUNGRY ──배고픔=0──▶ DEAD ──완료──▶ EGG
IDLE ──AI 요청──▶ THINKING ──응답──▶ TALKING ──닫기──▶ IDLE
스트레스=100 ──어느 상태에서든──▶ DEAD
```

### 상태별 시각 표현

| 상태 | 도형/색 표현 | 반복 |
|---|---|---|
| EGG | 흰 타원, 살짝 흔들림 | 무한 |
| HATCHING | 균열 애니메이션 → 캐릭터 등장 | 1회 |
| IDLE | 펫 색 원, 눈 깜빡임 | 무한 |
| HUNGRY | 주황 테두리 + 밥그릇 텍스트 | 무한 |
| DEAD | 회색, X눈, 쓰러짐 | 1회 |
| REACT_HAPPY | 노란 원 + "!" | 1회 |
| REACT_ANNOYED | 주황 원 + "..." | 1회 |
| REACT_ANGRY | 빨간 원 + ">_<" | 1회 |
| EATING | 초록 원 + "냠냠" | 1회 |
| WALKING | 펫 색, 좌우 이동·방향 반전 | 이동 중 |
| THINKING | 펫 색 + "..." 점 애니메이션 | API 대기 중 |
| TALKING | 펫 색 + 말풍선 활성 | 말풍선 열린 동안 |

## 4. 배고픔 시스템

| 항목 | 값 |
|---|---|
| 수치 범위 | 0~100 |
| 초기값 | 100 (부화 직후) |
| 감소율 | 10 / 1시간 (설정 변경 가능) |
| HUNGRY 진입 | 배고픔 < 70 |
| 추가 스트레스 | 배고픔 < 30 시 +3/시간 |
| 사망 | 배고픔 = 0 |

**먹이주기**
- 수치 +50 (최대 100), EATING 애니메이션 재생
- 배고픔 < 80일 때: 스트레스 -10
- 배고픔 ≥ 80일 때 (배부름): 먹긴 하지만 **스트레스 +10**

**저장**: 프로그램 종료 시 `pet_state.json` 기록. 재실행 시 경과 시간 계산하여 배고픔 차감.

## 5. 스트레스 시스템

| 항목 | 값 |
|---|---|
| 수치 범위 | 0~100 |
| 초기값 | 0 (부화 직후) |
| 사망 | 스트레스 = 100 |

### 증가 원인

| 원인 | 증가량 | 조건 |
|---|---|---|
| 방치 | +5 | 30분간 클릭·먹이주기·대화 없을 때마다 |
| 짜증 클릭 | +3/회 | REACT_ANNOYED 발생 시 |
| 화남 클릭 | +8/회 | REACT_ANGRY 발생 시 |
| 똥 방치 | +5/30분 | 똥 1개당 (누적) |
| 배고픔 심각 | +3/시간 | 배고픔 < 30 일 때 |
| 과식 | +10 | 배고픔 ≥ 80인데 먹이줄 때 |
| 나쁜 말 | +15 | 펫 대화 중 AI가 대화 맥락상 상처를 주는 말로 판단 시 |

### 감소 원인

| 원인 | 감소량 | 조건 |
|---|---|---|
| 적절한 클릭 | -5/회 | REACT_HAPPY 발생 시 |
| 먹이주기 | -10 | EATING 완료 시 |
| AI 대화 | -5 | 답변 수신 완료 시 |
| 똥 치워주기 | -8 | 똥 클릭 제거 시 |

### 단계별 시각 표현

| 수치 | 표현 |
|---|---|
| 0~30 | 기본 IDLE |
| 31~60 | 캐릭터 테두리 노랑, 가끔 "..." 말풍선 |
| 61~90 | 캐릭터 테두리 빨강, 자주 "ㅠㅠ" 말풍선 |
| 91~99 | 캐릭터 흔들림 + "저 힘들어요..." 말풍선 |
| 100 | DEAD 전환 |

## 6. 똥 처리 시스템

- **생성**: 랜덤 30분~2시간마다 펫 위치에 생성 (갈색 도형)
- **최대 개수**: 화면에 동시 3개
- **제거**: 똥 클릭 → 제거 + 스트레스 -8
- **방치 페널티**: 30분마다 똥 1개당 스트레스 +5
- **일괄 청소**: 우클릭 메뉴 → "청소"

## 7. 수치 확인 UI

펫 위에 마우스를 올리면 반투명 오버레이가 펫 바로 위에 나타남 (3초 후 자동 숨김).

```
┌─────────────────────┐
│ 배고픔: 72  ████░░ │
│ 스트레스: 45 ███░░░ │
└─────────────────────┘
        [펫]
```

- 배고픔 바: 정상 주황, 30 미만 시 빨강
- 스트레스 바: 정상 초록, 60 초과 시 빨강
- 마우스가 펫을 벗어나면 즉시 숨김

## 8. 이동 시스템

- 화면 전체를 자유롭게 배회
- 2~5초마다 랜덤 방향으로 1~3초 이동 (약 80px/초)
- 화면 가장자리 도달 시 방향 전환
- 드래그로 강제 이동 가능 (드래그 중 WALKING 정지, 완료 후 재개)
- 좌우 이동 시 캐릭터 방향 반전

### 이동 영역 제한 (사용자 정의 폴리곤)

- 우클릭 메뉴 → **이동 영역 설정** 으로 진입
- 전체화면 반투명 오버레이에서 좌클릭으로 꼭짓점을 순서대로 추가
- 꼭짓점 3개 이상 → **Enter** 또는 "완료" 버튼으로 확정
- **Backspace**: 마지막 꼭짓점 되돌리기 | **Esc**: 취소
- 확정된 폴리곤 안에서만 배회 (경계 이탈 시 새 방향 선택)
- 영역은 `config.json`에 저장되어 재실행 후에도 유지
- 우클릭 → **이동 영역 초기화** 로 전체 화면 배회 복귀

## 9. AI 대화 기능

### 펫 대화 모드
- 우클릭 → "펫에게 말 걸기" → 소형 입력창 팝업
- THINKING → 스트리밍 응답 → 말풍선 타이핑 효과
- 100자 이내 짧은 귀여운 답변 (시스템 프롬프트로 제한)
- 세션 내 대화 히스토리 유지 (멀티턴)
- 스트레스 -5
- AI가 대화 맥락상 주인의 말이 상처를 준다고 판단하면 스트레스 +15 (단순 키워드 매칭이 아닌 맥락 기반 판별)


**시스템 프롬프트**
```
당신은 PC-PET이라는 귀여운 데스크탑 펫입니다.
사용자를 "주인님"이라 부르고, 짧고 귀엽게 답변합니다 (100자 이내).
이모티콘을 적극 활용하세요.
현재 배고픔: {hunger}/100, 스트레스: {stress}/100
```

### 어시스턴트 모드
- 우클릭 → "AI 어시스턴트" → 확장 채팅창 (450×600px)
- 시스템 프롬프트 없음 (Claude 전체 능력: 코딩, 분석, 번역 등)
- 긴 응답 스크롤 표시
- 채팅창 열린 동안 TALKING 유지, 닫으면 IDLE 복귀

### 공통
- 기본 모델: `claude-haiku-4-5`
- API 키 없으면 AI 기능만 비활성화 (펫 시뮬레이션 정상 작동)
- API 키: console.anthropic.com 발급 → 설정창 1회 입력 → **OS 키체인에 암호화 저장** (Windows: Credential Manager, macOS: Keychain) — `keyring` 패키지 사용

## 15. 크로스 플랫폼 지원

### 지원 플랫폼
- **Windows 10/11** (주 타겟)
- **macOS 12+** (보조 지원)

### 호환성 분석

| 항목 | Windows | macOS |
|---|---|---|
| Python 3.11+ | ✅ | ✅ |
| PySide6 (Qt) | ✅ | ✅ |
| anthropic SDK | ✅ | ✅ |
| pywin32 | ✅ | ❌ (Windows 전용) |
| 투명 오버레이 | Win32 API | Cocoa/AppKit |
| 시스템 트레이 | 작업표시줄 아이콘 | 메뉴바 아이콘 |

### 플랫폼 추상화 설계

플랫폼별 코드는 `core/platform_utils.py`에 집중 관리. 나머지 코드는 이 모듈만 호출하여 플랫폼 무관하게 동작.

```python
# core/platform_utils.py
import platform

def set_click_through(window, enable: bool):
    """비캐릭터 영역 클릭 통과 설정"""
    if platform.system() == "Windows":
        _set_click_through_windows(window, enable)
    elif platform.system() == "Darwin":
        _set_click_through_mac(window, enable)

def _set_click_through_windows(window, enable):
    import win32gui, win32con
    hwnd = int(window.winId())
    style = win32gui.GetWindowLong(hwnd, win32con.GWL_EXSTYLE)
    if enable:
        style |= win32con.WS_EX_TRANSPARENT | win32con.WS_EX_LAYERED
    else:
        style &= ~win32con.WS_EX_TRANSPARENT
    win32gui.SetWindowLong(hwnd, win32con.GWL_EXSTYLE, style)

def _set_click_through_mac(window, enable):
    import ctypes, ctypes.util
    objc = ctypes.cdll.LoadLibrary(ctypes.util.find_library("objc"))
    # NSWindow.setIgnoresMouseEvents_(enable) 호출
    ns_window = objc.objc_msgSend(int(window.winId()), ...)
    objc.objc_msgSend(ns_window, "setIgnoresMouseEvents:", enable)
```

### 플랫폼별 패키지

```
# requirements.txt — 조건부 설치
pywin32==306 ; sys_platform == "win32"
```

macOS에서는 `pywin32` 설치 없이 ctypes로 Cocoa API 직접 호출.

### 시스템 트레이 차이
PySide6의 `QSystemTrayIcon`이 양 플랫폼을 모두 처리하므로 추가 작업 불필요.
macOS에서는 메뉴바 우측에 아이콘으로 표시됨.
